# 榫卯非遗文化传承平台 — 常见问题与答案

## 一、项目架构相关

### Q1：为什么项目要采用三层架构（Controller、Service、Mapper）？

**A：** 三层架构是 Java Web 开发的标准设计模式，各层职责如下：

| 层级 | 职责 | 作用 |
|------|------|------|
| Controller（控制层） | 接收 HTTP 请求，调用 Service，返回响应 | 前后端交互入口 |
| Service（业务层） | 处理业务逻辑，调用 Mapper | 核心业务规则 |
| Mapper（数据层） | 执行 SQL，与数据库交互 | 数据持久化 |

**好处：**
1. **职责分离**：每层只负责自己的工作，代码清晰
2. **便于维护**：修改某一层不会影响其他层
3. **便于测试**：可以单独测试 Service 层的业务逻辑
4. **便于复用**：Service 层的方法可以被多个 Controller 调用

---

### Q2：为什么 Service 层要定义接口（UserService）再写实现类（UserServiceImpl）？

**A：** 这是面向接口编程的思想，好处包括：
1. **降低耦合**：Controller 依赖接口而非实现类，便于切换实现
2. **便于扩展**：可以从 MySQL 切换到 MongoDB，只需修改实现类
3. **便于测试**：可以使用 Mock 对象进行单元测试
4. **符合 Spring AOP**：Spring 的代理机制基于接口实现

---

### Q3：MyBatis Plus 是什么？为什么要用它？

**A：** MyBatis Plus 是 MyBatis 的增强工具，在 MyBatis 基础上只做增强不做改变。

**主要功能：**
1. **通用 CRUD**：自动提供 insert、delete、update、select 等方法
2. **分页插件**：自动处理分页查询
3. **代码生成器**：自动生成 Entity、Mapper、Service、Controller
4. **条件构造器**：LambdaQueryWrapper 实现类型安全的查询

**使用好处：** 减少 80% 的重复代码，提高开发效率。

## 二、注解相关

### Q4：@RestController 和 @Controller 有什么区别？

**A：**
- **@Controller**：标识这是一个 Spring MVC 控制器，方法返回视图名称或需要配合 @ResponseBody 返回 JSON
- **@RestController**：是 @Controller + @ResponseBody 的组合，类中所有方法都返回 JSON 数据

**本项目使用 @RestController**，因为前后端分离，所有接口都返回 JSON。

---

### Q5：@Autowired 和 @Resource 有什么区别？

**A：**

| 特性 | @Autowired | @Resource |
|------|-----------|-----------|
| 来源 | Spring 注解 | Java 标准注解（JSR-250） |
| 注入方式 | 默认按类型注入 | 默认按名称注入 |
| 名称匹配 | 需要配合 @Qualifier | 直接通过 name 属性指定 |
| 兼容性 | 只能在 Spring 中使用 | 任何支持 JSR-250 的框架都能用 |

**本项目使用 @Resource**，因为按名称注入更直观，且是 Java 标准注解。

---

### Q6：@Transactional 注解的作用是什么？

**A：** @Transactional 用于声明事务管理，保证数据库操作的原子性。

**作用：**
1. 方法执行前开启事务
2. 方法正常执行后提交事务
3. 方法抛出异常时回滚事务

**使用场景：**
- 点赞/取消点赞（需要同时更新点赞记录和点赞数）
- 关注/取消关注（需要同时更新关注记录）
- 注册（需要同时创建用户和初始化相关数据）

**参数说明：**
- `rollbackFor = Exception.class`：对所有异常都回滚（默认只对 RuntimeException 回滚）

## 三、数据库相关

### Q7：数据库脚本执行报错 `The used table type doesn't support FULLTEXT indexes` 怎么办？

**A：** 这是因为 MySQL 版本低于 5.6.4，不支持 InnoDB 引擎的 FULLTEXT 索引。

**解决方案：**
1. **升级 MySQL** 到 5.6.4 或更高版本（推荐）
2. **删除 FULLTEXT 索引**：将 `FULLTEXT KEY` 改为普通 `KEY`
3. **改用 MyISAM 引擎**：将 `ENGINE=InnoDB` 改为 `ENGINE=MyISAM`（不推荐，会丢失事务支持）

---

### Q8：DBeaver 连接 MySQL 失败怎么办？

**A：** 检查以下几点：
1. **MySQL 服务是否启动**：在命令行执行 `net start mysql`（Windows）或 `sudo service mysql start`（Linux）
2. **驱动是否正确**：在 DBeaver 中下载对应版本的 MySQL 驱动
3. **连接配置是否正确**：检查主机、端口、用户名、密码
4. **防火墙是否拦截**：确保 3306 端口开放

---

### Q9：为什么密码要加密存储？使用的是什么加密算法？

**A：** 密码加密是为了安全：
1. 防止数据库泄露时密码被直接获取
2. 即使管理员也无法查看用户的明文密码
3. 符合安全规范

**本项目使用 MD5 加密**，通过 `DigestUtils.md5DigestAsHex()` 实现。

**注意**：实际项目中建议使用更安全的 **BCrypt** 算法，因为它自带盐值，安全性更高。

---

## 四、Spring Boot 相关

### Q10：为什么访问 `localhost:8081` 显示 Whitelabel Error Page 404？

**A：** 这是正常现象，因为：
1. 本项目是 **纯 REST API 后端**，没有配置首页页面
2. 根路径 `/` 没有对应的 Controller 处理方法
3. 所有 API 接口都以 `/api` 开头

**验证方法：** 访问 `http://localhost:8081/api/banner/list`，如果返回 JSON 数据，说明服务正常。

---

### Q11：如何修改服务端口？

**A：** 修改 `application.yml` 文件：

```yaml
server:
  port: 8081  # 修改为你想要的端口
```

---

### Q12：Spring Boot 内嵌的 Tomcat 版本是多少？和外部 Tomcat 8.5 冲突吗？

**A：** Spring Boot 2.7.18 内嵌的是 **Tomcat 9.0.83**。

**不冲突**，因为：
1. 内嵌 Tomcat 只在直接运行 JAR 时使用
2. 如果部署到外部 Tomcat，需要将打包方式改为 WAR，并排除内嵌 Tomcat
3. Spring Boot 2.7.x 兼容 Servlet 3.1/4.0 规范，Tomcat 8.5 也支持

## 五、微信小程序相关

### Q13：微信小程序请求后端接口报错 `fail url not in domain list` 怎么办？

**A：** 这是因为微信小程序有域名白名单限制。

**解决方案：**
1. **开发阶段**：在微信开发者工具中，点击 `详情` → `本地设置` → 勾选 `不校验合法域名`
2. **生产阶段**：在小程序管理后台的 `开发管理` → `开发设置` → `服务器域名` 中添加后端域名

---

### Q14：微信小程序如何访问本地后端服务？

**A：** 由于微信小程序不允许访问 `localhost`，需要：
1. 使用局域网 IP 代替 localhost（如 `http://192.168.1.100:8081`）
2. 手机和电脑连接同一 WiFi
3. 在微信开发者工具中开启 `不校验合法域名`

---

## 六、代码相关

### Q15：Result 类的作用是什么？为什么要统一响应格式？

**A：** Result 是统一响应结果封装类，作用：
1. **统一格式**：所有接口返回相同的 JSON 结构
2. **便于前端处理**：前端可以用统一的方式解析响应
3. **包含状态信息**：code 表示状态码，message 表示提示信息，data 表示数据

**结构：**
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

---

### Q16：什么是多态关联？项目中哪里用到了？

**A：** 多态关联是指一个表可以关联到多个不同的表。

**本项目中的应用：**
- **comment 表**：通过 `target_type`（1：文章、2：帖子）和 `target_id` 关联到 article 或 post
- **like_record 表**：通过 `target_type`（1：文章、2：帖子）和 `target_id` 关联到 article 或 post

**好处：** 避免为每种内容类型单独创建评论表或点赞表，减少表数量。

---

### Q17：@MapperScan 和 @Mapper 有什么区别？

**A：**

| 特性 | @MapperScan | @Mapper |
|------|-------------|---------|
| 作用范围 | 扫描整个包 | 标记单个接口 |
| 使用位置 | 启动类上 | Mapper 接口上 |
| 便利性 | 一次配置，扫描所有 Mapper | 每个 Mapper 都需要标记 |
| 灵活性 | 统一配置 | 单独控制 |

**本项目同时使用**：启动类上用 @MapperScan 扫描整个包，每个 Mapper 接口上保留 @Mapper 作为良好习惯。

## 七、部署相关

### Q18：如何在 IDEA 中运行项目？

**A：**
1. 打开 IDEA，导入 `ljx_backend` 项目
2. 等待 Maven 依赖下载完成
3. 找到 `LjxPlatformApplication.java`
4. 右键 → `Run 'LjxPlatformApplication'`
5. 观察控制台，确认 `Tomcat started on port(s): 8081`

---

### Q19：如何打包部署到外部 Tomcat？

**A：**
1. 修改 `pom.xml`，将 `<packaging>jar</packaging>` 改为 `<packaging>war</packaging>`
2. 添加 `spring-boot-starter-tomcat` 依赖，scope 设为 `provided`
3. 修改启动类，继承 `SpringBootServletInitializer`
4. 执行 `mvn package` 打包
5. 将生成的 WAR 文件放入 Tomcat 的 `webapps` 目录
6. 启动 Tomcat

---

### Q20：项目使用的 Java 版本是多少？可以用 Java 11 或 17 吗？

**A：** 本项目使用 **Java 1.8**。

**可以升级到 Java 11 或 17**，但需要：
1. 修改 `pom.xml` 中的 `java.version`
2. 确保所有依赖兼容新版本
3. 测试所有功能正常

**注意**：Spring Boot 2.7.x 支持 Java 8-19，Spring Boot 3.x 需要 Java 17+。

---

## 八、开发相关

### Q21：如何添加一个新的 API 接口？

**A：** 按照三层架构的流程：

1. **Entity 层**：定义数据实体（如需要新表）
2. **Mapper 层**：定义数据访问接口，继承 `BaseMapper<Entity>`
3. **Service 层**：
   - 定义接口，继承 `IService<Entity>`
   - 实现类，继承 `ServiceImpl<Mapper, Entity>`，实现接口
4. **Controller 层**：定义控制器，注入 Service，编写接口方法

**示例**：添加一个查询用户收藏文章列表的接口
1. 在 `ArticleService` 接口中添加方法
2. 在 `ArticleServiceImpl` 中实现方法
3. 在 `ArticleController` 中添加接口方法

---

### Q22：如何进行单元测试？

**A：** 使用 Spring Boot 的测试框架：

```java
@SpringBootTest
public class UserServiceTest {

    @Autowired
    private UserService userService;

    @Test
    public void testGetById() {
        User user = userService.getById(1);
        assertNotNull(user);
        assertEquals("测试用户", user.getNickname());
    }
}
```

---

### Q23：如何查看 SQL 执行日志？

**A：** 在 `application.yml` 中配置：

```yaml
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

这样可以在控制台看到执行的 SQL 语句。

## 九、安全相关

### Q24：项目中有哪些安全措施？

**A：**
1. **密码加密**：使用 MD5 加密存储密码
2. **跨域配置**：CorsConfig 配置允许的来源和方法
3. **统一异常处理**：GlobalExceptionHandler 隐藏敏感错误信息
4. **事务管理**：@Transactional 保证数据一致性

---

### Q25：如何防止 SQL 注入？

**A：** 本项目使用 MyBatis Plus，通过 `#{}` 占位符自动进行参数转义，防止 SQL 注入。

**注意**：不要使用 `${}` 拼接 SQL，这会导致 SQL 注入风险。

---

## 十、性能相关

### Q26：如何优化数据库查询性能？

**A：**
1. **添加索引**：在经常查询的字段上添加索引
2. **分页查询**：避免一次性查询大量数据
3. **缓存**：使用 Redis 缓存热点数据
4. **连接池**：使用 Druid 连接池管理数据库连接

---

### Q27：项目使用了哪些连接池？

**A：** 使用 **Druid** 连接池，配置在 `application.yml` 中：

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
```

Druid 是阿里巴巴开源的数据库连接池，具有监控、防 SQL 注入等功能。
